Ejercicio 1

Al crear un proyecto nuevo, Android Studio te muestra el porcentaje de
dispositivos que tu aplicación alcanzará según el Minimum SDK elegido. Si elegir un API
Level antiguo (como el 21) te da un alcance de casi el 100%, ¿cuál sería la desventaja técnica
o de desarrollo de no elegir siempre la versión más moderna disponible? 

Elegir un API Level antiguo limita el acceso a nuevas APIs, herramientas modernas y mejoras de rendimiento y seguridad, lo que obliga a usar código más complejo o soluciones alternativas, aumentando la dificultad de mantenimiento.

Ejercicio 2

Android obliga a separar los textos de la interfaz en el archivo strings.xml en
lugar de escribirlos directamente en el código Kotlin. Más allá de la traducción a otros
idiomas (internacionalización), menciona otra ventaja técnica de esta práctica relacionada
con el mantenimiento y la escalabilidad de una aplicación a largo plazo

Centralizar los textos en strings.xml permite modificar contenidos sin tocar el código fuente, facilitando mantenimiento, reutilización y escalabilidad al evitar duplicación de textos en múltiples archivos.
